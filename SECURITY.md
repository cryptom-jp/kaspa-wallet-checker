# Security Policy

このリポジトリのセキュリティポリシーと、脆弱性を発見した場合の連絡方法をまとめています。

## 対象範囲

このポリシーは、次の内容を対象とします。

- このリポジトリに含まれるソースコード
- このリポジトリで提供している設定ファイルやスクリプト
- リポジトリに関連する公開ドキュメント

以下の内容は原則として対象外です（ただし、重大なリスクがある場合は歓迎します）。

- このリポジトリ外のサービス設定（取引所アカウント、クラウド環境など）
- サードパーティライブラリ自体の既知の脆弱性

## 脆弱性を見つけた場合の連絡方法

セキュリティ上の問題を発見した場合は、**公開の Issue ではなく、非公開の手段**でご連絡ください[web:139][web:142]。

1. GitHub のセキュリティタブに「Security policy」が表示されていれば、そちらの案内に従ってください。
2. もしくは、次の情報を含めて、リポジトリオーナーの連絡先（例：GitHubプロフィールに記載のメールアドレス）までご連絡ください。

記載していただきたい情報の例[web:134][web:137][web:139]：

- 脆弱性の種類（例：情報漏えいの可能性、認証回避、権限昇格など）
- 影響範囲（どの機能 / どのファイルに影響するか）
- 再現手順（可能な範囲で）
- PoC（Proof of Concept：再現用の簡単なコードや手順）があれば大変助かります
- 既に公開されている問題かどうか（すでにSNSやブログ等で公開している場合は、そのURL）

可能な限り速やかに確認し、必要に応じて修正や対応方針を検討します。

## 公開 Issue / Pull Request について

セキュリティ上の問題を含む内容は、以下の理由から **公開 Issue では報告しないでください**。

- 他のユーザーが悪用する前に対策を行うため
- APIキーや個人情報などが誤って公開されることを避けるため

修正案の Pull Request を送る場合は、**脆弱性の詳細そのものは PR の本文に書かず**、別途プライベートに連絡をお願いします。

## シークレット情報の取り扱い

このリポジトリでは、次のような情報はコミットしない運用とします。

- ウォレットのシードフレーズ、秘密鍵
- 取引所やAPIサービスの APIキー / シークレット
- パスワードや個人情報
- `.env` などの環境変数ファイルの中身

もしこれらを誤ってコミットしてしまった場合は：

1. 対象のキーを **すぐに無効化 / ローテーション** してください。
2. 過去のコミット履歴からも削除することを検討してください。
3. 必要に応じて、Issue ではなくプライベートチャンネルを通じてオーナーに連絡してください。

## 貢献してくれる方へのお願い

- 新しい機能や変更を提案する際は、セキュリティへの影響（権限、入力値検証、ログ出力など）も一緒に考えていただけると非常に助かります。
- セキュリティに関する改善提案（設計の改善、ライブラリアップデートなど）も歓迎します。

---

このポリシーは、プロジェクトの成長や運用状況に応じて更新される場合があります。
